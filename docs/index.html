04 — Intuition

Martial law vs. local police

Martial law (direct computation)

A central command collects every signal from every intersection in the city, computes all $2^n$ possible configurations, then issues one master order. Works fine for 3–4 intersections. Becomes physically impossible for 50.

Local police (factor graph)

Each officer only watches their own intersection and talks to immediate neighbours. From thousands of these small local interactions, a coherent city-wide traffic pattern emerges — with no one ever seeing the full picture.

Local rule
Each factor encodes one small compatibility rule between a few neighbouring variables — nothing more.
Message passing
A variable's "belief" is built only from messages sent by its direct factor neighbours — never from the global state.
Global emergence
After a few rounds, every node holds the exact marginal or optimal assignment — identical to brute-force enumeration, at a fraction of the cost.
Scalability
Add more intersections — the centralised method collapses, but the factor graph's per-node cost never changes.
